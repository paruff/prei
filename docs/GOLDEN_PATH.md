# Golden Path — prei

> The single lowest-friction route from idea to deployed feature.
> Every agent, every PM follows this path.

---

## The Path at a Glance

```
1. SPEC      PM writes issue using .github/ISSUE_TEMPLATE/feature.md
2. BRANCH    git checkout -b feat/ISSUE-ID-description
3. TEST      Write failing pytest test → commit (RED)
4. IMPLEMENT Prompt Copilot with AC + failing test → commit (GREEN)
5. VERIFY    ruff check . && black --check . && pytest -q (all must pass)
6. PR        Draft PR with AI-Assisted Review Block
7. CI GATE   All gates pass
8. REVIEW    @review-agent → human review (< 24h target)
9. MERGE     Squash merge: feat(scope): [description] (#N)
10. DEPLOY   Merge to main triggers CI/CD
```

## Financial Feature Pattern

For any new KPI calculation:

1. Add the pure function to `finance/utils.py`
2. Write unit tests with: happy path, negative inputs, zero inputs, very large values
3. Call from `core/services/` (not directly from views)
4. Expose via `core/api_views.py` with DRF serializer validation
5. Add entry to `docs/API_SURFACE.md`

```python
# ✅ Correct pattern
# finance/utils.py — pure function
def calculate_cap_rate(noi: Decimal, purchase_price: Decimal) -> Decimal:
    """Calculate Cap Rate as NOI / Purchase Price.
    
    Args:
        noi: Net Operating Income (must be >= 0)
        purchase_price: Property purchase price (must be > 0)
    
    Returns:
        Cap rate as a decimal (e.g. 0.08 for 8%)
    
    Raises:
        ValueError: If purchase_price is zero or negative
    """
    if purchase_price <= 0:
        raise ValueError(f"purchase_price must be > 0, got {purchase_price}")
    return noi / purchase_price

# ❌ Wrong — financial math in a view
class PropertyDetailView(APIView):
    def get(self, request, pk):
        prop = Property.objects.get(pk=pk)
        cap_rate = prop.noi / prop.purchase_price  # NEVER
```

## Migration Pattern

Every database migration needs:
1. Forward migration (auto-generated by Django)
2. A note in the PR about the rollback path
3. Tested against a real Postgres instance in CI (not just SQLite)

---

## Off-Path (Requires Human Judgment First)

| Situation | Why |
|---|---|
| New `requirements.txt` dependency | Security review, license |
| New database migration | Rollback path, production impact |
| Changing authentication/permission logic | Security-critical |
| New external API integration | Key management, rate limits, cost |
| Change touching > 5 files | Break into multiple issues |
