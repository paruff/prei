## Why

passive real estate investors need a single pane of glass for foreclosure and REO opportunities across multiple data sources. Before v0.4.0, prei only supported VRM foreclosure listings. Adding HUD, USDA, county, and ATTOM data meant building a reusable discovery source model layer — then wiring six integrations on top of it.

## What

v0.4.0 adds structured ingestion for **HUD REO** (real ArcGIS Hub GeoJSON endpoint), **USDA REO** (fixed-width TXT parser from data.gov), **Dallas County TX NTS** (Playwright-based public records scraper), and **ATTOM preforeclosure notices** (ZIP-based API integration). Each source feeds into a standardized `CountyForeclosureNotice` or source-specific model, then flows through the existing pipeline screening and "Add to Pipeline" workflow.

## How

The release is built around the DISC (Discovery Integration) increment framework. Each DISC adds one source:

- **DISC-0**: Model layer — `HudProperty`, `UsdaProperty`, `CountyForeclosureNotice`
- **DISC-1/DISC-2**: HUD and USDA REO ingestion via management commands
- **DISC-3/DISC-4**: Pipeline screening and list/detail views for source properties
- **DISC-5/DISC-6**: Dallas County scraper and ATTOM preforeclosure integration

The stack runs Python 3.14 + Django 6.0, with Playwright handling JavaScript-rendered content and ATTOM providing structured API data.

## Proof

All 14 ATTOM preforeclosure tests pass (mocked), all 15 Dallas County scraper tests pass (mocked), and all 10 HUD/USDA ingestion tests pass. The full test suite runs clean on CI with coverage above 80%.
